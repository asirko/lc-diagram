Pourquoi SVG ?

REX :

Comment faire un gros formulaire pour construire un parcours de formation avec de conditions ?
Pas facile avec un formulaire, un diagramme serait plus adapté.
D3.js semblait un bon candidat, mais pas simple à prendre en main.
Il se trouve qu'SVG est plutôt simple à prendre en main, et s'utilise bien avec Angular.

Par contre SVG ne fait pas le café ! C'est pas pour tout.
Suivant votre besoin, HTML ou Canvas seront plus adaptés.

HTML
  • tout développeur est à l'aise avec HTML 🎓
  • gestion événementiel (parfaitement intégré à Angular)
  • CSS très puissant
  • formes trop basiques
  • très peu adapté pour le dessin 😖 (courbes?, flèches?)
  • peu performant pour un grand nombre d'objets graphiques
SVG
  • XML facile à prendre en main
  • c'est fait pour du dessin 💕
  • gestion événementiel (bien intégré à Angular)
  • CSS plutôt bien supporté
  • interopérable 🚀 (outils de dessin, lib comme Lottie, export SVG)
  • temps d'apprentissage (viewBox, arcs de cercle)
  • CSS partiellement supporté (pas de z-index 😢)
  • peu performant pour un grand nombre d'objets graphiques
Canvas
  • meilleur en performance
  • adapté pour du dessin
  • très polyvalent
  • outilage : WebGL ou ThreeJS 🤟
  • temps d'apprentissage
  • tout est à déclarer en JavaScript, peut devenir verbeux
  • outils natifs très bas niveau 😰 (pas facile de faire de la détection de clic, les flèches... , pour l'a10n vous êtes seul)
